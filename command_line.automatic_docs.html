wfl#

Workflow command line interface.

wfl [OPTIONS] COMMAND [ARGS]...

Options

-v, --verbose#

descriptor#

wfl descriptor [OPTIONS] COMMAND [ARGS]...

quippy#

wfl descriptor quippy [OPTIONS]

Options

--local#

calculate a local (per-atom) descriptor

--force#

overwrite existing info or arrays item if present

--descriptor <descriptor>#

Required quippy.descriptors.Descriptor arg string

--key <key>#

Required Atoms.info (global) or Atoms.arrays (local) for descriptor vector

-i, --inputs <inputs>#

Required Input xyz file(s) to create ConfigSet from

-o, --outputs <outputs>#

Required Ouput file to create OutputSpec from.

ace#

evaluates ACE

wfl eval ace [OPTIONS]

Options

-i, --inputs <inputs>#

Required Input xyz file(s) to create ConfigSet from

-o, --outputs <outputs>#

Required Ouput file to create OutputSpec from.

--param-fname <param_fname>#

Path to the potential parameter file

-p, --prop-prefix <prop_prefix>#

Prefix to be pre-pended to all evaluate properties. Defaults to “gap_”/”ace_”/”mace_” as appropriate

--num-inputs-per-python-subprocess <num_inputs_per_python_subprocess>#

Number of configs to be evaluated per each calculator initialization

Default

10

generate#

wfl generate [OPTIONS] COMMAND [ARGS]...

buildcell#

Repeatedly runs buildcell (from Pickard’s AIRSS distribution) to generate random configs with specified species, volumes, distances, symmetries, etc.

Minimal contents of –buildcell-input file:

#TARGVOL=<min_vol>-<max_vol> (NOTE: volume is volume_per_formula_unit/number_of_species)
#SPECIES=<elem_symbol_1>%NUM=<num_1>[,<elem_symbol_2>%NUM=<num_2 …]
#NFORM=[ <n_min>-<n_max> | { <n_1>, <n_2>, … } ]
#SYMMOPS=<n_min>-<n_max> (NOTE: optional)
#SLACK=0.25
#OVERLAP=0.1
#COMPACT
#MINSEP=<min_separation_default> <elem_symbol_1>-<elem_symbol_1>=<min_separation_1_1> [
<elem_symbol_1>-<elem_symbol_2=<min_separation_1_2> … ]
##EXTRA_INFO <info_field>=<value> (NOTE: optional)
wfl generate buildcell [OPTIONS]

Options

--buildcell-input <buildcell_input>#

Required buildcell input file

--buildcell-exec <buildcell_exec>#

Required buildcell executable including path

-N, --n-configs <n_configs>#

Required number of configs to generate

--perturbation <perturbation>#

magnitude of random perturbation to atomic positions

-o, --outputs <outputs>#

Required Ouput file to create OutputSpec from.

-i, --extra-info <extra_info>#

Extra key=val pairs to add to Atoms.info

select#

wfl select [OPTIONS] COMMAND [ARGS]...

cur#

Select structures by CUR

wfl select cur [OPTIONS]

Options

-N, --n-configs <n_configs>#

Required number of configs to select

--keep_descriptor#

keep the descriptor value in the final config file

--kernel_exponent <kernel_exponent>#

exponent of dot-product for kernel

--deterministic#

use deterministic (not stochastic) CUR selection

--key <key>#

Required Atoms.info (global) or Atoms.arrays (local) for descriptor vector

--stochastic-seed <stochastic_seed>#

seed for np.random.seed() in stochastic CUR.

-i, --inputs <inputs>#

Required Input xyz file(s) to create ConfigSet from

-o, --outputs <outputs>#

Required Ouput file to create OutputSpec from.

lambda#

selects atoms based on a lambda function

wfl select lambda [OPTIONS]

Options

-e, --exec-code <exec_code>#

Required String to be evaluated by the lambda function. Will be substituted into eval(“lambda atoms: ” + exec_code).

-i, --inputs <inputs>#

Required Input xyz file(s) to create ConfigSet from

-o, --outputs <outputs>#

Required Ouput file to create OutputSpec from.
