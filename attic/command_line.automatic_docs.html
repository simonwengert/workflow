wfl#

GAP workflow command line interface.

wfl [OPTIONS] COMMAND [ARGS]...

Options

-v, --verbose#

descriptor#

wfl descriptor [OPTIONS] COMMAND [ARGS]...

calc#

wfl descriptor calc [OPTIONS] [INPUTS]...

Options

--output-file <output_file>#

Required

--descriptor <descriptor>#

Required quippy.Descriptor arg string

--key <key>#

Required key to store in Atoms.info (global) or Atoms.arrays(local)

--local#

calculate a local (per-atom) descriptor

--force#

overwrite existing info or arrays item if present

INPUTS#

generate-configs#

wfl generate-configs [OPTIONS] COMMAND [ARGS]...

repeat-buildcell#

Repeatedly runs buildcell (from Pickard’s AIRSS distribution) to generate random configs with specified species, volumes, distances, symmetries, etc.

Minimal contents of –buildcell-input file:

#TARGVOL=<min_vol>-<max_vol> (NOTE: volume is volume_per_formula_unit/number_of_species)
#SPECIES=<elem_symbol_1>%NUM=<num_1>[,<elem_symbol_2>%NUM=<num_2 …]
#NFORM=[ <n_min>-<n_max> | { <n_1>, <n_2>, … } ]
#SYMMOPS=<n_min>-<n_max> (NOTE: optional)
#SLACK=0.25
#OVERLAP=0.1
#COMPACT
#MINSEP=<min_separation_default> <elem_symbol_1>-<elem_symbol_1>=<min_separation_1_1> [
<elem_symbol_1>-<elem_symbol_2=<min_separation_1_2> … ]
##EXTRA_INFO <info_field>=<value> (NOTE: optional)
wfl generate-configs repeat-buildcell [OPTIONS]

Options

--output-file <output_file>#
--buildcell-input <buildcell_input>#

Required buildcell input file

--buildcell-exec <buildcell_exec>#

Required buildcell executable including path

-N, --n-configs <n_configs>#

Required number of configs to generate

--extra-info <extra_info>#

dict of information to store in Atoms.info

--perturbation <perturbation>#

magnitude of random perturbation to atomic positions

committee#

Calculated energy and force with a committee of models. Uses a prefix on the filenames, this can be later changed does not support globbed filenames as xyz input

configset works in many -> many mode here

wfl processing committee [OPTIONS] [INPUTS]...

Options

-p, --prefix <prefix>#

Prefix to put on the first

-g, --gap-fn <gap_fn>#

Required gap filenames, globbed

-f, --force#

force writing

Arguments

INPUTS#

Optional argument(s)

reference-error#

wfl processing reference-error [OPTIONS] [INPUTS]...

Options

-pre, --pre-calc <pre_calc>#

Required string to exec() that sets up for calculator, e.g. imports

-c, --calc <calc>#

Required string to ‘eval()’ that returns calculator constructor

-a, --calc-args <calc_args>#

json list of calculator constructor args

-k, --calc-kwargs <calc_kwargs>#

json dict of calculator constructor kwargs

-r, --ref-prefix <ref_prefix>#

string to prepend to info/array keys for reference energy, forces, virial. If None, use SinglePointCalculator results

-p, --properties <properties>#

command separated list of properties to use

-C, --category_keys <category_keys>#

comma separated list of info keys to use for doing per-category error

-o, --outfile <outfile>#

output file, - for stdout

--intermed-file <intermed_file>#

intermediate file to contain calculator results, keep in memory if None

INPUTS#
